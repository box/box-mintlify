Every API call requires an **Access Token** to identify the authenticated user.
For security purposes, Access Tokens expire after 60 minutes. If you are using
<Link href="/guides/authentication/oauth2">OAuth 2.0</Link>, use the provided <Link href="/guides/authentication/tokens/refresh/">Refresh Token</Link> to obtain a new Access
Token. If you are using server authentication, <Link href="/guides/authentication/jwt">JWT</Link> or
<Link href="/guides/authentication/client-credentials">Client Credentials Grant</Link>, make an API call to the <Link href="/reference/post-oauth2-token">token endpoint</Link>
to request a new Access Token.

## OAuth 2.0

If your application leverages <Link href="/guides/authentication/oauth2">OAuth 2.0</Link> for authentication, you can
follow the steps below to obtain a token pair via <Link href="/guides/tooling/postman/install">Postman</Link>.

<Frame border center shadow>
  ![OAuth2.0 token request](/images/guides/tooling/postman/oauth-postman.png)
</Frame>

- The `grant_type` will always be `authorization_code`.
- The `client_id` and `client_secret` values can be obtained from the **Configuration** tab for your application in the [Developer Console][dc].

To obtain the value for `code`, build and visit your
<Link href="/guides/authentication/oauth2/without-sdk">authorization URL</Link> in your browser. Complete the OAuth 2.0 flow and,
upon redirecting to your configured redirect URL, the authorization code will be
at the end of the URL. As a reminder, this authorization code is only valid for
30 seconds. This means you must enter it to the designated field in Postman and
click **Send** before it expires. Therefore, we recommend entering the other
values so the API call is ready to send as soon as you get the code.
