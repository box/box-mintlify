---
title: "Archive"
---

API version 2025.0

An archive is a folder dedicated to storing content that is
redundant, outdated, or trivial. Content in an archive is not accessible to its
owner and collaborators.
To use this feature, you must request GCM scope for your Box application.

<ParamField body="id" type="string">
example: `12345`

The unique identifier that represents an archive.
</ParamField>

<ParamField body="type" type="string">
example: `archive`

The value is always `archive`.

Value is always `archive`
</ParamField>

<ParamField body="description" type="string">
example: `This is an archive for important documents.`

The description of the archive.
</ParamField>

<ParamField body="name" type="string">
example: `Archive`

The name of the archive.

The following restrictions to the archive name apply: names containing
non-printable ASCII characters, forward and backward slashes
(`/`, `\`), names with trailing spaces, and names `.` and `..` are
not allowed.
</ParamField>

<ParamField body="owned_by" type="object">
example: `{"id":"12345","type":"user"}`

The part of an archive API response that describes the user who owns the archive.

<Expandable>
  <ParamField body="id" type="string">
  example: `12345`
  
  The unique identifier that represents a user who owns the archive.
  </ParamField>

  <ParamField body="type" type="string">
  example: `user`
  
  The value is always `user`.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="size" type="integer(int64)">
example: `123456789`

The size of the archive in bytes.
</ParamField>
