<ParamField body="days_valid" type="integer">
example: `2`

Set the number of days after which the
created signature request will automatically
expire if not completed. By default, we do
not apply any expiration date on signature
requests, and the signature request does not expire.
</ParamField>

<ParamField body="email_message" type="string">
example: `Hello! Please sign the document below`

Message to include in signature request email. The field
is cleaned through sanitization of specific characters. However,
some html tags are allowed. Links included in the
message are also converted to hyperlinks in the email. The
message may contain the following html tags including `a`, `abbr`,
`acronym`, `b`, `blockquote`, `code`, `em`, `i`, `ul`, `li`, `ol`, and
`strong`. Be aware that when the text
to html ratio is too high, the email
may end up in spam filters. Custom styles on
these tags are not allowed.
If this field is not passed, a default message will be used.
</ParamField>

<ParamField body="email_subject" type="string">
example: `Sign Request from Acme`

Subject of signature request email. This is cleaned by sign
request. If this field is not passed, a default subject will be used.
</ParamField>

<ParamField body="name" type="string">
example: `Official contract`

The name of the template.
</ParamField>

<ParamField body="parent_folder" type="Folder (Mini)object">
The destination folder to place final, signed document and signing
log. Only `ID` and `type` fields are required. The root folder,
folder ID `0`, cannot be used.
</ParamField>

<ParamField body="ready_sign_link" type="object">
Box's ready-sign link feature enables you to create a
link to a signature request that
you've created from a template. Use this link
when you want to post a signature request
on a public form — such as an email, social media post,
or web page — without knowing who the signers will be.
Note: The ready-sign link feature is
limited to Enterprise Plus customers and not
available to Box Verified Enterprises.

<ParamField body="signers" type="object[]">
Array of signers for the template.

**Note**: It may happen that some signers specified in the template belong to conflicting [segments](/reference/resources/shield-information-barrier-segment-member/) (user groups).
This means that due to the security policies, users are assigned to segments to prevent exchanges or communication that could lead to ethical conflicts.
In such a case, an attempt to send a sign request based on a template that lists signers in conflicting segments will result in an error.

Read more about [segments and ethical walls](https://support.box.com/hc/en-us/articles/9920431507603-Understanding-Information-Barriers#h_01GFVJEHQA06N7XEZ4GCZ9GFAQ).

<Expandable>
  <ParamField body="email" type="string">
  example: `example@mail.com`
  
  Email address of the signer.
  </ParamField>

      Relative y coordinate to the page the input is on, ranging from 0 to 1.

    </Expandable>

    Date prefill value.

    <Expandable>
      <ParamField body="height" type="number">

      Relative height to the page the input is on, ranging from 0 to 1.
      </ParamField>
    
      <ParamField body="width" type="number">
      example: `0.2618657937806874`
      
      Relative width to the page the input is on, ranging from 0 to 1.
      </ParamField>
    
    </Expandable>
    </ParamField>
  
    <ParamField body="document_id" type="string">

    Document identifier.
    </ParamField>
  
    <ParamField body="document_tag_id" type="string">
    example: `1234`
    
    This references the ID of a specific tag contained in a file of the signature request.
    </ParamField>
  
    <ParamField body="dropdown_choices" type="stringarray">
    example: `["Yes","No","Maybe"]`
    
    When the input is of the type `dropdown` this
    values will be filled with all the
    dropdown options.
    </ParamField>
  
    <ParamField body="group_id" type="string">
    example: `da317330-225a-4c72-89ad-0d6dcaaf4df6`
    
    When the input is of type `radio` they can be
    grouped to gather with this identifier.
    </ParamField>
  
    <ParamField body="is_required" type="boolean">
    example: `true`
    
    Whether or not the input is required.
    </ParamField>
  
    <ParamField body="label" type="string">
    example: `Legal name`
    
    The label field is used especially for text, attachment, radio, and checkbox type inputs.
    </ParamField>
  
    <ParamField body="page_index" type="integer">
    example: `4`
    
    Index of page that the input is on.
    </ParamField>
  
    <ParamField body="read_only" type="boolean">
    example: `true`
    
    Whether this input was defined as read-only(immutable by signers) or not.
    </ParamField>
  
    <ParamField body="text_value" type="string">
    example: `text`
    
    Text prefill value.
    </ParamField>
  
  </Expandable>
  </ParamField>

  <ParamField body="is_in_person" type="boolean">
  example: `true`
  
  Used in combination with an embed URL for a sender.
  After the sender signs, they will be
  redirected to the next `in_person` signer.
  </ParamField>

  <ParamField body="is_password_required" type="boolean">
  example: `true`
  
  If true for signers with a defined email, the password provided when the template was created is used by default.
  If true for signers without a specified / defined email, the creator needs to provide a password when using the template.
  </ParamField>

  <ParamField body="is_phone_number_required" type="boolean">
  example: `true`
  
  If true for signers with a defined email, the phone number provided when the template was created is used by default.
  If true for signers without a specified / defined email, the template creator needs to provide a phone number when creating a request.
  </ParamField>

  <ParamField body="label" type="string">
  example: `Jane Doe`
  
  A placeholder label for the signer set by the template creator to differentiate between signers.
  </ParamField>

  <ParamField body="login_required" type="boolean">
  example: `true`
  
  If true, the signer is required to login to access the document.
  </ParamField>

  <ParamField body="order" type="integer">
  example: `2`
  
  Order of the signer.
  </ParamField>

  <ParamField body="public_id" type="string">
  example: `RJZYYVPR`
  
  An identifier for the signer. This can be used to identify a signer within the template.
  </ParamField>

  <ParamField body="role" type="string">
  example: `signer`
  
  Defines the role of the signer in the signature request. A role of
  `signer` needs to sign the document, a role `approver`
  approves the document and
  a `final_copy_reader` role only
  receives the final signed document and signing log.
  
  Value is one of `signer`,`approver`,`final_copy_reader`
  </ParamField>

  <ParamField body="signer_group_id" type="string">
  example: `cd4ff89-8fc1-42cf-8b29-1890dedd26d7`
  
  If provided, this value points signers that are assigned the same inputs and belongs to same signer group.
  A signer group is not a Box Group. It is an entity that belongs to the template itself and can only be used
  within Box Sign requests created from it.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="source_files" type="File (Mini)array">
List of files to create a signing document from.
Only the ID and type fields are required
for each file.
</ParamField>
