title: "Collaboration"
---

API version 2024.0

<Note>
This resource is used by endpoints in the version **2024.0**.
For more details, see **[Box API versioning](/guides/api-calls/api-versioning-strategy/)**. 

Learn more about [Box SDK versioning strategy](/guides/tooling/sdks/sdk-versioning).
</Note>

Collaborations define access permissions for users and groups to files and
folders, similar to access control lists. A collaboration object grants a
user or group access to a file or folder with permissions defined by a
specific role.

<ParamField body="id" type="string">
example: `12345678`

The unique identifier for this collaboration.
</ParamField>

<ParamField body="type" type="string">
example: `collaboration`

The value will always be `collaboration`.

Value is always `collaboration`
</ParamField>

<ParamField body="acceptance_requirements_status" type="object">

<Expandable>
  <ParamField body="strong_password_requirement" type="object">
  
  <Expandable>
    <ParamField body="enterprise_has_strong_password_required_for_external_users" type="boolean">
    example: `true`
    
    Whether or not the enterprise that owns the content requires
    a strong password to collaborate on the content, or enforces
    an exposed password detection for the external collaborators.
    </ParamField>
  
    <ParamField body="user_has_strong_password" type="boolean">
    example: `true`
    
    Whether or not the user has a strong and not exposed password set
    for their account. The field is `null` when a strong password is
    not required.
    </ParamField>
  
  </Expandable>
  </ParamField>

  <ParamField body="terms_of_service_requirement" type="object">
  
  <Expandable>
    <ParamField body="is_accepted" type="boolean">
    example: `true`
    
    Whether or not the terms of service have been accepted. The
    field is `null` when there is no terms of service required.
    </ParamField>
  
    <ParamField body="terms_of_service" type="Terms of service (Base)object">
    The terms of service that must be accepted before the
    collaboration can be accepted. The field is `null` when
    there is no terms of service required.
    </ParamField>
  
  </Expandable>
  </ParamField>

  <ParamField body="two_factor_authentication_requirement" type="object">
  
  <Expandable>
    <ParamField body="enterprise_has_two_factor_auth_enabled" type="boolean">
    example: `true`
    
    Whether or not the enterprise that owns the content requires
    two-factor authentication to be enabled in order to
    collaborate on the content.
    </ParamField>
  
    <ParamField body="user_has_two_factor_authentication_enabled" type="boolean">
    example: `true`
    
    Whether or not the user has two-factor authentication
    enabled. The field is `null` when two-factor
    authentication is not required.
    </ParamField>
  
  </Expandable>
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="accessible_by" type="Group (Mini)/User (Collaborations)object">
The user or group that is granted access.
</ParamField>

<ParamField body="acknowledged_at" type="string(date-time)">
example: `2012-12-12T10:55:20-08:00`

When the `status` of the collaboration object changed to
`accepted` or `rejected`.
</ParamField>

<ParamField body="app_item" type="App itemobject">
An `app_item` to which access is granted. The field is `null`
when the collaboration is created on an item (see `item`
field), or the `app_item` is inaccessible. The role cascades
to all items associated with the `app_item`.
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

When the collaboration object was created.
</ParamField>

<ParamField body="created_by" type="User (Collaborations)object">
example: `[{"id":33224412},{"type":"user"},{"login":"dylan@example.com"},{"name":"Dylan Smith"}]`

The user who created the collaboration object.
</ParamField>

<ParamField body="expires_at" type="string(date-time)">
example: `2012-12-26T10:53:43-08:00`

When the collaboration will expire, or `null` if no expiration
date is set.
</ParamField>

<ParamField body="invite_email" type="string">
example: `john@example.com`

The email address used to invite an unregistered collaborator, if
they are not a registered user.
</ParamField>

<ParamField body="is_access_only" type="boolean">
example: `true`

If set to `true`, collaborators have access to
shared items, but such items won't be visible in the
All Files list. Additionally, collaborators won't
see the path to the root folder for the
shared item.
</ParamField>

<ParamField body="item" type="File/Folder/Web linkobject">
The file or folder to which access is granted. The field is
`null` when the collaboration `status` is `pending` or the
collaboration is created on an app item (see `app_item` field).
</ParamField>

<ParamField body="modified_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

When the collaboration object was last modified.
</ParamField>

<ParamField body="role" type="string">
example: `editor`

The level of access granted.

Value is one of `editor`,`viewer`,`previewer`,`uploader`,`previewer uploader`,`viewer uploader`,`co-owner`,`owner`
</ParamField>

<ParamField body="status" type="string">
example: `accepted`

The status of the collaboration invitation. If the status
is `pending`, `login` and `name` return an empty string.

Value is one of `accepted`,`pending`,`rejected`
</ParamField>
