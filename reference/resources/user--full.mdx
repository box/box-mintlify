API version 2024.0

<Note>
This resource is used by endpoints in the version **2024.0**.
For more details, see **[Box API versioning](/guides/api-calls/api-versioning-strategy/)**. 

Learn more about [Box SDK versioning strategy](/guides/tooling/sdks/sdk-versioning).

A full representation of a user, as can be returned from any
user API endpoint.

* [Full](/reference/resources/user--full/)
* [Standard](/reference/resources/user/)
* [Mini](/reference/resources/user--mini/)
* [Base](/reference/resources/user--base/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **full** variant can be returned
by API endpoints that support the `fields` parameter. For example,
by defining the `fields` request parameter as
`id,type` when requesting a file by ID, only those fields will
be returned in the API response.
</Info>

<ParamField body="id" type="string">
example: `11446498`

The unique identifier for this user.
</ParamField>

<ParamField body="type" type="string">
example: `user`

The value will always be `user`.

Value is always `user`
</ParamField>

<ParamField body="address" type="string">
example: `900 Jefferson Ave, Redwood City, CA 94063`

The user’s address.
</ParamField>

<ParamField body="avatar_url" type="string">
example: `https://www.box.com/api/avatar/large/181216415`

URL of the user’s avatar image.
</ParamField>

<ParamField body="can_see_managed_users" type="boolean">
example: `true`

Whether the user can see other enterprise users in their contact list.
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

When the user object was created.
</ParamField>

<ParamField body="enterprise" type="object">
Representation of the user’s enterprise.

  example: `11446498`

  </ParamField>

  <ParamField body="type" type="string">

  <ParamField body="name" type="string">

  </ParamField>

</ParamField>

<ParamField body="external_app_user_id" type="string">
example: `my-user-1234`

An external identifier for an app user, which can be used to look up
the user. This can be used to tie user IDs from external identity
providers to Box users.
</ParamField>

<ParamField body="hostname" type="string">
example: `https://example.app.box.com/`

The root (protocol, subdomain, domain) of any links that need to be
generated for the user.
</ParamField>

<ParamField body="is_exempt_from_device_limits" type="boolean">
example: `true`

Whether to exempt the user from Enterprise device limits.
</ParamField>

<ParamField body="is_exempt_from_login_verification" type="boolean">
example: `true`

Whether the user must use two-factor authentication.
</ParamField>

<ParamField body="is_external_collab_restricted" type="boolean">
example: `true`

Whether the user is allowed to collaborate with users outside their
enterprise.
</ParamField>

<ParamField body="is_platform_access_only" type="boolean">
example: `true`

Whether the user is an App User.
</ParamField>

<ParamField body="is_sync_enabled" type="boolean">
example: `true`

Whether the user can use Box Sync.
</ParamField>

<ParamField body="job_title" type="string">
example: `CEO`

The user’s job title.
</ParamField>

<ParamField body="language" type="string">
example: `en`

The language of the user, formatted in modified version of the
[ISO 639-1](/guides/api-calls/language-codes/) format.
</ParamField>

<ParamField body="login" type="string(email)">
example: `ceo@example.com`

The primary email address of this user.
</ParamField>

<ParamField body="max_upload_size" type="integer(int64)">
example: `2147483648`

The maximum individual file size in bytes the user can have.
</ParamField>

<ParamField body="modified_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

When the user object was last modified.
</ParamField>

<ParamField body="my_tags" type="stringarray">
example: `["important"]`

Tags for all files and folders owned by the user. Values returned
will only contain tags that were set by the requester.
</ParamField>

<ParamField body="name" type="string">
example: `Aaron Levie`

The display name of this user.
</ParamField>

<ParamField body="notification_email" type="object">
An alternate notification email address to which email
notifications are sent. When it's confirmed, this will be
the email address to which notifications are sent instead of
to the primary email address.

<Expandable>
  <ParamField body="email" type="string">
  example: `notifications@example.com`
  
  The email address to send the notifications to.
  </ParamField>

  <ParamField body="is_confirmed" type="boolean">
  example: `true`
  
  Specifies if this email address has been confirmed.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="phone" type="string">
example: `6509241374`

The user’s phone number.
</ParamField>

<ParamField body="role" type="string">
example: `admin`

The user’s enterprise role.

Value is one of `admin`,`coadmin`,`user`
</ParamField>

<ParamField body="space_amount" type="integer(int64)">
example: `11345156112`

The user’s total available space amount in bytes.
</ParamField>

<ParamField body="space_used" type="integer(int64)">
example: `1237009912`

The amount of space in use by the user.
</ParamField>

<ParamField body="status" type="string">
example: `active`

The user's account status.

Value is one of `active`,`inactive`,`cannot_delete_edit`,`cannot_delete_edit_upload`
</ParamField>

<ParamField body="timezone" type="string(timezone)">
example: `Africa/Bujumbura`

The user's timezone.
</ParamField>

<ParamField body="tracking_codes" type="objectarray">
Tracking codes allow an admin to generate reports from the
admin console and assign an attribute to a specific group
of users. This setting must be enabled for an enterprise
before it can be used.

<Expandable>
  <ParamField body="type" type="string">
  example: `tracking_code`
  
  The value will always be `tracking_code`.
  
  Value is always `tracking_code`
  </ParamField>

  <ParamField body="name" type="string">
  example: `department`
  
  The name of the tracking code, which must be preconfigured in
  the Admin Console.
  </ParamField>

  <ParamField body="value" type="string">
  example: `Sales`
  
  The value of the tracking code.
  </ParamField>

</Expandable>
</ParamField>
