A user representation for integration mappings
API purposes. Fields name and login are not required.

* [Full](/reference/resources/user-integration-mappings-reference--full/)
* [Standard](/reference/resources/user-integration-mappings-reference/)
* [Mini](/reference/resources/user-integration-mappings-reference--mini/)
* [Base](/reference/resources/user-integration-mappings-reference--base/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **base** variant represent
the default fields that are always returned when using the `fields`
query parameter, additional to the fields specified in the
parameter.
</Info>

<ParamField body="id" type="string">
example: `11446498`

The unique identifier for this user.
</ParamField>

<ParamField body="type" type="string">
example: `user`

The value will always be `user`.

Value is always `user`
</ParamField>

<ParamField body="login" type="string(email)">
example: `ceo@example.com`

The primary email address of this user.
</ParamField>

<ParamField body="name" type="string">
example: `Aaron Levie`

The display name of this user.
</ParamField>

  "login": "ceo@example.com",

}

</ResponseExample>
