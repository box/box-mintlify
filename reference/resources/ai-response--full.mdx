AI ask response.

* [Full](/reference/resources/ai-response--full/)
* [Standard](/reference/resources/ai-response/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **full** variant can be returned
by API endpoints that support the `fields` parameter. For example,
by defining the `fields` request parameter as
`id,type` when requesting a file by ID, only those fields will
be returned in the API response.
</Info>

<ParamField body="ai_agent_info" type="object">
The information on the models and processors used in the request.

<Expandable>
  <ParamField body="models" type="objectarray">
  The models used for the request.
  
  <Expandable>
    <ParamField body="name" type="string">
    example: `azure__openai__text_embedding_ada_002`
    
    The name of the model used for the request.
    </ParamField>
  
    <ParamField body="provider" type="string">
    example: `azure`
    
    The provider that owns the model used for the request.
    </ParamField>
  
    <ParamField body="supported_purpose" type="string">
    example: `embedding`
    
    The supported purpose utilized by the model used for the request.
    </ParamField>
  
  </Expandable>
  </ParamField>

  <ParamField body="processor" type="string">
  example: `basic_text`
  
  The processor used for the request.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="answer" type="string">
example: `Public APIs are important because of key and important reasons.`

The answer provided by the LLM.
</ParamField>

<ParamField body="citations" type="objectarray">
The citations of the LLM's answer reference.

<Expandable>
  <ParamField body="id" type="string">
  example: `123`
  
  The id of the item.
  </ParamField>

  <ParamField body="type" type="string">
  example: `file`
  
  The type of the item.
  
  Value is always `file`
  </ParamField>

  <ParamField body="content" type="string">
  example: `Public APIs are key drivers of innovation and growth.`
  
  The specific content from where the answer was referenced.
  </ParamField>

  <ParamField body="name" type="string">
  example: `The importance of public APIs.pdf`
  
  The name of the item.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="completion_reason" type="string">
example: `done`

The reason the response finishes.
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

The ISO date formatted timestamp of when the answer to the prompt was created.
</ParamField>
