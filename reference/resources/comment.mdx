API version 2024.0



Standard representation of a comment.

* [Full](/reference/resources/comment--full/)
* [Standard](/reference/resources/comment/)
* [Base](/reference/resources/comment--base/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **standard** variant are generally
returned when the resource is requested via its own API endpoints.
For example, when retrieving a file by ID, it will return
these fields unless the `fields` parameter has been specified.
</Info>

<ParamField body="id" type="string">
example: `11446498`

The unique identifier for this comment.
</ParamField>

<ParamField body="type" type="string">
example: `comment`

The value will always be `comment`.

Value is always `comment`
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

The time this comment was created.
</ParamField>

<ParamField body="created_by" type="User (Mini)object">
A mini user object representing the author of the
comment.
</ParamField>

<ParamField body="is_reply_comment" type="boolean">
example: `true`

Whether or not this comment is a reply to another
comment.
</ParamField>

<ParamField body="item" type="object">
The file this comment was placed on.

<Expandable>
  <ParamField body="id" type="string">
  example: `11446498`
  
  The unique identifier for this object.
  </ParamField>

  <ParamField body="type" type="string">
  example: `file`
  
  The type for this object.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="message" type="string">
example: `@Aaron Levie these tigers are cool!`

The text of the comment, as provided by the user.
</ParamField>

<ParamField body="modified_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

The time this comment was last modified.
</ParamField>
