---
title: "User (Collaborations)"
---

API version 2024.0



A mini representation of a user, can be returned only when
the status is `pending`.

* [Full](/reference/resources/user-collaborations--full/)
* [Standard](/reference/resources/user-collaborations/)
* [Mini](/reference/resources/user-collaborations--mini/)
* [Base](/reference/resources/user-collaborations--base/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **base** variant represent
the default fields that are always returned when using the `fields`
query parameter, additional to the fields specified in the
parameter.
</Info>

<ParamField body="id" type="string">
example: `11446498`

The unique identifier for this user.
</ParamField>

<ParamField body="type" type="string">
example: `user`

The value will always be `user`.

Value is always `user`
</ParamField>

<ParamField body="is_active" type="boolean">
example: `true`

If set to `false`, the user is either deactivated or deleted.
</ParamField>

<ParamField body="login" type="string(email)">
example: `ceo@example.com`

The primary email address of this user. If the collaboration status is `pending`, an empty string is returned.
</ParamField>

<ParamField body="name" type="string">
example: `Aaron Levie`

The display name of this user. If the collaboration status is `pending`, an empty string is returned.
</ParamField>

<ResponseExample>

```json Response Example
{
  "id": "11446498",
  "type": "user",
  "is_active": true,
  "login": "ceo@example.com",
  "name": "Aaron Levie"
}
```

</ResponseExample>
