Represents a configured webhook.

* [Standard](/reference/resources/webhook/)
* [Mini](/reference/resources/webhook--mini/)

<Info>
This resource has a few variations that can be encountered when
using the API.

The fields that are part of the **standard** variant are generally
returned when the resource is requested via its own API endpoints.
For example, when retrieving a file by ID, it will return
these fields unless the `fields` parameter has been specified.
</Info>

<ParamField body="id" type="string">
example: `11446498`

The unique identifier for this webhook.
</ParamField>

<ParamField body="type" type="string">
example: `webhook`

The value will always be `webhook`.

Value is always `webhook`
</ParamField>

<ParamField body="address" type="string">
example: `https://example.com/webhooks`

The URL that is notified by this webhook.
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2012-12-12T10:53:43-08:00`

A timestamp identifying the time that
the webhook was created.
</ParamField>

<ParamField body="created_by" type="User (Mini)object">
The user who created the webhook.
</ParamField>

<ParamField body="target" type="object">
The item that will trigger the webhook.

<Expandable>
  <ParamField body="id" type="string">
  example: `1231232`
  
  The ID of the item to trigger a webhook.
  </ParamField>

  <ParamField body="type" type="string">
  example: `file`
  
  The type of item to trigger a webhook.
  
  Value is one of `file`,`folder`
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="triggers" type="stringarray">
example: `["FILE.UPLOADED"]`

An array of event names that this webhook is
to be triggered for.
</ParamField>
