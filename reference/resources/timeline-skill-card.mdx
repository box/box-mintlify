---
title: "Timeline Skill Card"
---

API version 2024.0



A Box Skill metadata card that places a list of images on a
timeline.

<ParamField body="type" type="string">
example: `skill_card`

The value will always be `skill_card`.

Value is always `skill_card`
</ParamField>

<ParamField body="created_at" type="string(date-time)">
example: `2018-04-13T13:53:23-07:00`

The optional date and time this card was created at.
</ParamField>

<ParamField body="duration" type="integer">
example: `1000`

An total duration in seconds of the timeline.
</ParamField>

<ParamField body="entries" type="object[]">
A list of entries on the timeline.

<Expandable>
  <ParamField body="appears" type="object[]">
  Defines a list of timestamps for when this item should appear on the
  timeline.
  
  <Expandable>
    <ParamField body="end" type="integer">
    example: `20`
    
    The time in seconds when an
    entry should stop appearing on a timeline.
    </ParamField>
  
    <ParamField body="start" type="integer">
    example: `1`
    
    The time in seconds when an
    entry should start appearing on a timeline.
    </ParamField>
  
  </Expandable>
  </ParamField>

  <ParamField body="image_url" type="string">
  example: `https://example.com/image1.jpg`
  
  The image to show on a for an entry that appears
  on a timeline. This image URL is required for every entry.
  
  The image will be shown in a
  list of items (for example faces), and clicking
  the image will show the user where that entry
  appears during the duration of this entry.
  </ParamField>

  <ParamField body="text" type="string">
  example: `John`
  
  The text of the entry. This would be the display
  name for an item being placed on the timeline, for example the name
  of the person who was detected in a video.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="invocation" type="object">
The invocation of this service, used to track
which instance of a service applied the metadata.

<Expandable>
  <ParamField body="id" type="string">
  example: `image-recognition-service-123`
  
  A custom identifier that represent the instance of
  the service that applied this metadata. For example,
  if your `image-recognition-service` runs on multiple
  nodes, this field can be used to identify the ID of
  the node that was used to apply the metadata.
  </ParamField>

  <ParamField body="type" type="string">
  example: `skill_invocation`
  
  The value will always be `skill_invocation`.
  
  Value is always `skill_invocation`
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill" type="object">
The service that applied this metadata.

<Expandable>
  <ParamField body="id" type="string">
  example: `image-recognition-service`
  
  A custom identifier that represent the service that
  applied this metadata.
  </ParamField>

  <ParamField body="type" type="string">
  example: `service`
  
  The value will always be `service`.
  
  Value is always `service`
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill_card_title" type="object">
The title of the card.

<Expandable>
  <ParamField body="code" type="string">
  example: `Faces`
  
  An optional identifier for the title.
  </ParamField>

  <ParamField body="message" type="string">
  example: `Faces`
  
  The actual title to show in the UI.
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill_card_type" type="string">
example: `timeline`

The value will always be `timeline`.

Value is always `timeline`
</ParamField>
