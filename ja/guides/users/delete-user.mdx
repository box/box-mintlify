App Userと管理対象ユーザーの削除プロセスは同じです。ユーザーアカウントを削除するには、削除するアカウントのユーザーIDを指定します。

<CodeGroup>

```sh cURL
curl -i -X DELETE "https://api.box.com/2.0/users/12345" \
     -H "authorization: Bearer <ACCESS_TOKEN>"
```

```typescript Node/TypeScript v10
await client.users.deleteUserById(user.id);
```

```python Python v10
client.users.delete_user_by_id(user.id)
```

```cs .NET v10
await client.Users.DeleteUserByIdAsync(userId: user.Id);
```

```swift Swift v10
try await client.users.deleteUserById(userId: user.id)
```

```java Java v10
client.getUsers().deleteUserById(user.getId())
```

```java Java v5
BoxUser user = new BoxUser(api, "0");
user.delete(false, false);
```

```python Python v4
user_id = '33333'
client.user(user_id).delete(force=True)
```

```cs .NET v6
await client.UsersManager.DeleteEnterpriseUserAsync("44444", notify: false, force: true);
```

```javascript Node v4
// Delete the user even if they still have files in their account
client.users.delete('123', {force: true})
    .then(() => {
        // deletion succeeded — no value returned
    });
```

</CodeGroup>

ユーザーアカウントを削除するときに設定できるオプションパラメータも2つあります。

* force: アカウントにまだコンテンツがある場合でも、ユーザーを削除するかどうか。
* notify: アカウントが削除されたという通知をユーザーに送信するかどうか。

<Note>

ユーザーアカウントにまだコンテンツがある場合、ユーザー削除リクエストは失敗します。これを解決するには、別のアカウントに[すべてのファイルまたはフォルダを転送する](/ja/guides/users/deprovision/transfer-folders)か、オプションの`force`パラメータを使用します。

</Note>
