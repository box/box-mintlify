title: "ステータススキルカード"
---

詳細については、
**[Box APIのバージョン管理](/ja/guides/api-calls/api-versioning-strategy/)**を参照してください。

「[Box SDKのバージョニング戦略](/ja/guides/tooling/sdks/sdk-versioning/)について詳しく学ぶ。」
</Note>

メタデータサイドバーにステータスメッセージを配置するBox Skillメタデータカード。

<ParamField body="type" type="string">
例 `skill_card`

値は常に`skill_card`になります。

次の値に固定: `skill_card`
</ParamField>

<ParamField body="created_at" type="string(date-time)">
例 `2018-04-13T13:53:23-07:00`

このカードの作成日時 (省略可)。
</ParamField>

<ParamField body="invocation" type="object">
このサービスの呼び出し。メタデータを適用したサービスのインスタンスを追跡するために使用されます。

<Expandable>
  <ParamField body="id" type="string">
  例 `image-recognition-service-123`
  
  このメタデータを適用したサービスのインスタンスを表すカスタム識別子。たとえば、`image-recognition-service`が複数のノードで動作する場合、このフィールドを使用すると、メタデータの適用に使用されたノードのIDを識別できます。
  </ParamField>

  <ParamField body="type" type="string">
  例 `skill_invocation`
  
  値は常に`skill_invocation`になります。
  
  次の値に固定: `skill_invocation`
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill" type="object">
このメタデータを適用したサービス。

<Expandable>
  <ParamField body="id" type="string">
  例 `image-recognition-service`
  
  このメタデータを適用したサービスを表すカスタム識別子。
  </ParamField>

  <ParamField body="type" type="string">
  例 `service`
  
  値は常に`service`になります。
  
  次の値に固定: `service`
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill_card_title" type="object">
カードのタイトル。

<Expandable>
  <ParamField body="code" type="string">
  例 `status`
  
  タイトルの識別子 (省略可)。
  </ParamField>

  <ParamField body="message" type="string">
  例 `Status`
  
  UI上で表示する実際のタイトル。
  </ParamField>

</Expandable>
</ParamField>

<ParamField body="skill_card_type" type="string">
例 `status`

値は常に`status`になります。

次の値に固定: `status`
</ParamField>

<ParamField body="status" type="object">
スキルのステータスを設定します。これは、スキルがデータを処理している間またはファイルを処理できなかった場合にユーザーにメッセージを表示するのに使用できます。

<Expandable>
  <ParamField body="code" type="string">
  例 `success`
  
  このスキル呼び出しのステータスを表すコード。デフォルトでは、各コードには独自のメッセージが伴います。これらのメッセージを調整するには、このオブジェクトの`message`値を設定します。
  
  次の値のいずれか1つ: `invoked`,`processing`,`success`,`transient_failure`,`permanent_failure`
  </ParamField>

  <ParamField body="message" type="string">
  例 `We're preparing to process your file. Please hold on!`
  
  このステータスで提供できるカスタムメッセージ。ウェブアプリでエンドユーザーに表示されます。
  </ParamField>

</Expandable>
</ParamField>

<ResponseExample>

```json レスポンスの例
{
  "type": "skill_card",
  "created_at": "2018-04-13T13:53:23-07:00",
  "invocation": {
    "id": "image-recognition-service-123",
    "type": "skill_invocation"
  },
  "skill": {
    "id": "image-recognition-service",
    "type": "service"
  },
  "skill_card_title": {
    "code": "status",
    "message": "Status"
  },
  "skill_card_type": "status",
  "status": {
    "code": "success",
    "message": "We're preparing to process your file. Please hold on!"
  }
}
```

</ResponseExample>
