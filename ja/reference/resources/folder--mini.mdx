---
title: "フォルダ (Mini)"
---

APIバージョン 2024.0


他のリソースの下にネストされたときに使用されるファイルバージョンのMini版の表示。

* [Full](/ja/reference/resources/folder--full/)
* [Standard](/ja/reference/resources/folder/)
* [Mini](/ja/reference/resources/folder--mini/)
* [Base](/ja/reference/resources/folder--base/)

<Info>
このリソースにはAPIの使用時に発生する可能性があるいくつかのバリエーションがあります。

通常、**Mini**バリアントに含まれるフィールドは、このリソースが別のリソースのレスポンス内にネストされている場合に返されます。
</Info>

<ParamField body="id" type="string">
例 `12345`

フォルダを表す一意の識別子。

フォルダIDを確認するには、ウェブアプリケーションでフォルダにアクセスして、URLからIDをコピーします。たとえば、URLが`https://*.app.box.com/folders/123`の場合、`folder_id`は`123`です。
</ParamField>

<ParamField body="type" type="string">
例 `folder`

値は常に`folder`になります。

次の値に固定: `folder`
</ParamField>

<ParamField body="etag" type="string">
例 `1`

このフォルダのHTTP `etag`。これは変更が発生した場合 (またはしなかった場合) にフォルダに対して変更を行う目的でのみ、`If-Match`および`If-None-Match`ヘッダー内の一部のAPIエンドポイントで使用できます。
</ParamField>

<ParamField body="name" type="string">
例 `Contracts`

フォルダの名前。
</ParamField>

<ParamField body="sequence_id" type="string">
例 `3`

この項目に適用された最新のUser Eventを表す数値の識別子。

これを`GET /events`エンドポイントと組み合わせて使用すると、この識別子が読み取られる前に発生した可能性があるUser Eventを除外できます。

たとえば、Box DriveなどのアプリケーションがAPIを介して項目を取得し、その項目の変更に関連するUser Eventの発生を監視する場合などがこれに該当します。User Eventの`sequence_id`が最初に取得されたリソースの`sequence_id`よりも小さいか同じである場合、アプリケーションはそのようなUser Eventをすべて無視します。
</ParamField>

<ResponseExample>

```json レスポンスの例
{
  "id": "12345",
  "type": "folder",
  "etag": "1",
  "name": "Contracts",
  "sequence_id": "3"
}
```

</ResponseExample>
